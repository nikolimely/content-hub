# Content Hub — CLAUDE.md

## Project Overview
Content Hub is an internal content management dashboard for Limely. It manages articles across multiple sites, handles AI content generation, publishes to GitHub repos, and schedules posts via Vercel deploy hooks.

Live at: `https://content.limely.co.uk`
Repo: `git@github.com:nikolimely/content-hub.git`

## Tech Stack
- **Framework**: Next.js 16 (App Router, Turbopack)
- **Database**: Neon (PostgreSQL) via `@prisma/adapter-neon` HTTP adapter
- **ORM**: Prisma 7 — config in `prisma.config.ts` (not schema.prisma)
- **AI**: Anthropic Claude (`@anthropic-ai/sdk`) + OpenAI (`gpt-4o-mini`)
- **Editor**: Tiptap WYSIWYG
- **Auth**: Magic link via `jose` (JWT) + Resend for email
- **Email**: Resend — only `@limely.co.uk` addresses allowed
- **Styling**: Tailwind v4, dark theme (`#0f0f0f` bg, `#2A2944` sidebar)
- **Brand**: Limely — sidebar uses `#2A2944` navy + `#A7C838` lime accent

## Key Architecture

### Database (Prisma 7 + Neon)
- Connection URL goes in `prisma.config.ts` datasource, NOT in `schema.prisma`
- `schema.prisma` only has `provider = "postgresql"` (no url field)
- `lib/db.ts` uses `PrismaNeonHttp` adapter (HTTP, not WebSocket — required for Vercel serverless)
- `dotenv` is loaded in `prisma.config.ts` so `DATABASE_URL` is available for migrations locally
- Run migrations: `npx prisma migrate deploy`
- Regenerate client: `npx prisma generate`

### Authentication (Magic Link)
- `lib/auth.ts` — JWT signing/verification (magic tokens: 15min, sessions: 7d)
- `middleware.ts` — protects all routes except `/login` and `/api/auth/*`
- Session stored in `content_hub_session` httpOnly cookie
- Only `@limely.co.uk` emails can sign in (hardcoded in `/api/auth/request/route.ts`)
- Login page matches Limely brand: white bg, `#2a2f3f` card, Limely logo, `#A7C838` button

### Content Pipeline
1. **Planned** → article exists in DB, no content yet
2. **Draft** → content generated by AI, not yet published
3. **Published** → committed to GitHub repo as MDX file
   - **Live**: `scheduledAt <= now` (or null)
   - **Scheduled**: `scheduledAt > now` (file in GitHub but filtered by date on the static site)

### Scheduled Post Auto-Publishing
- Cron at `/api/cron/publish` runs **hourly** (Vercel Cron)
- Finds sites with posts due in the last hour, triggers their Vercel deploy hook
- Deploy hook URL stored per-site in `Site.deployHook` field
- Cron at `/api/cron/sync` runs **daily at 6am** — syncs from GitHub CMS

### Publishing to GitHub
- `lib/publish.ts` — commits MDX files to the site's GitHub repo
- `lib/github.ts` — GitHub API helpers (get/put files, list directories)
- Images are committed to `assetsPath` (e.g. `public/images/blog`)
- After publish, `scheduledAt` is set from frontmatter `date` if future-dated

### Preview Mode (Dynamically)
- Content Hub redirects to `https://{site.domain}/api/enable-draft?secret=...&slug=...&articleId=...`
- Dynamically validates secret, enables Next.js draft mode, redirects to article
- Article page fetches live content from Content Hub's `/api/preview/[articleId]` endpoint
- `PREVIEW_SECRET` must match between both projects
- `CONTENT_HUB_URL` env var on Dynamically points to Content Hub

### Sync from GitHub
- `lib/sync.ts` — reads `cms/settings.md`, `cms/content-plan.md`, and content directories
- Syncs authors from `authorsPath` file (e.g. `src/lib/authors.ts`)
- Backfills `scheduledAt` from frontmatter `date` for all existing articles

## Environment Variables (Vercel)
```
DATABASE_URL        Neon PostgreSQL connection string
GITHUB_TOKEN        GitHub PAT for repo read/write
ANTHROPIC_API_KEY   For Claude models
OPENAI_API_KEY      For GPT models
AUTH_SECRET         JWT signing secret (32+ chars)
RESEND_API_KEY      Resend.com API key
PREVIEW_SECRET      Shared with Dynamically for draft preview
CRON_SECRET         Vercel cron auth header secret
```

## Local Development
- Runs on port 2222: `npm run dev`
- Local DB: SQLite at `prisma/dev.db` (dev only — production uses Neon)
- `DATABASE_URL` in `.env` points to Neon for running migrations locally
- SSH key for GitHub pushes: `~/.ssh/github`

## Site: Dynamically
- Repo: `Limely/dynamically`
- Content path: `content/posts`
- URL path: `/insights`
- Authors file: `src/lib/authors.ts`
- Deploy hook: set in Dynamically site settings in Content Hub

## Migrations
When adding new fields:
1. Add to `prisma/schema.prisma`
2. Create `prisma/migrations/{timestamp}_{name}/migration.sql` manually (PostgreSQL syntax)
3. Run `npx prisma migrate deploy`
4. Run `npx prisma generate`
